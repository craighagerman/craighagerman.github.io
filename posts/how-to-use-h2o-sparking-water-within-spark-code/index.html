How to Use H2o Sparking Water Within Spark Code

The Situation

I had an existing Spark (Scala) project (using Spark ML algorithms). I wanted to try out H2O Sparking Water algorithms within this project. The problem is that most documentation around H2O Sparkling Water is are different use cases.

For example, many examples start a sparking-shell:

cd sparkling-water-3.32.0.4-1-2.4
bin/sparkling-shell

or specify Sparking Water when starting a spark-shell:

$SPARK_HOME/bin/spark-shell --packages ai.h2o:sparkling-water-package_2.11:3.32.0.4-1-2.4

On the other had I already have 36k lines of Scala Spark code. Adding Sparking Water dependencies and importing relevant libraries didn’t work (various errors thrown). The following is what I did to get things working.

Versioning

It is necessary to choose Sparking Water versions that match your Spark and Scala versions. This is what I am using:

Spark: 3.0.1
Scala: 2.12.10 (OpenJDK)

n.b. Naming of Sparkling Water releases follows Spark release. e.g.

  • 1.4.x is for Spark 1.4
  • 3.x is for Spark 3.x

Gradle Dependencies

I use gradle as a dependency manager for this Scala project. I discovered that the following three dependencies are required:

compile "org.apache.spark:spark-repl_2.12:3.1.0"
compile "ai.h2o:sparkling-water-package_2.12:3.32.0.4-1-3.0"
compile group: 'org.scala-lang', name: 'scala-compiler', version: '2.12.10'

The sparkling-water-package dependency is obvious. You need Sparking Water …. to run Sparking Water algorithms. (n.b. For this kind of use case you need the package dependency, not core etc)

But the two other dependencies are non-obvious.

spark-repl is needed to prevent java.lang.NoClassDefFoundError: org/apache/spark/repl/Main errors.

see https://sparkbyexamples.com/h2o-sparkling-water/sparkling-water-java-lang-noclassdeffounderror-scala-tools-nsc-interpreter-interactivereader/

scala-compiler is needed to prevent java.lang.NoClassDefFoundError: scala/tools/nsc/interpreter/InteractiveReader errors.

see: https://sparkbyexamples.com/h2o-sparkling-water/sparkling-water-java-lang-noclassdeffounderror-org-apache-spark-repl-main/

Test Code

Use FunSpec and SharedSparkSessionBase to create a test spec (edit package <com.companyname>.ml.clustering as needed) :

package <com.companyname>.ml.clustering

import ai.h2o.sparkling.H2OContext
import ai.h2o.sparkling.ml.algos.H2OKMeans
import org.apache.spark.sql.functions.col
import org.apache.spark.sql.test.SharedSparkSessionBase
import org.apache.spark.sql.{DataFrame, SparkSession}
import org.scalatest.FunSpec

class H20ClusteringDemoSpec extends FunSpec with SharedSparkSessionBase {

  private val dataDir = "smalldata"

  /* Load specified data file from $dataDir resources directory*/
  private def getDataFile(fileName: String) = {
    getClass.getResource(s"/${dataDir}/${fileName}").getFile
  }

  def getProstateData(spark: SparkSession): DataFrame = {
    val dataFile = getDataFile("prostate.csv")
    spark.read.format("csv").option("header", "true").option("inferSchema", "true").load(dataFile)
  }

  def getIrisData(spark: SparkSession): DataFrame = {
    val dataFile = getDataFile("iris_wheader.csv")
    spark.read.format("csv").option("header", "true").option("inferSchema", "true").load(dataFile)
  }

  describe("Demo H20 Algorithms") {
    it("Run H2O KMeans on prostate data") {
      val df = getProstateData(spark)
      // Select a subsample
      val train = df.filter(col("Capsule") === 1)
      H2OContext.getOrCreate
      val kmeans = new H2OKMeans().setK(3)
      val model = kmeans.fit(train)
      val prediction = model.transform(train).select("prediction").collect()
      println(prediction.mkString("\n===> Model predictions: ", ", ", ", ...\n"))
    }

    it("Run H2O KMeans on iris data") {
      val df = getIrisData(spark)
      H2OContext.getOrCreate
      val kmeans = new H2OKMeans().setK(3)
      val model = kmeans.fit(df)
      val prediction = model.transform(df).select("prediction").collect()
      println(prediction.mkString("\n===> Model predictions: ", ", ", ", ...\n"))
    }
  }
}
